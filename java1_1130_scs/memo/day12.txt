1. 사용자 정의 타입
	ex) int num = 10;
	    클래스명 객체명 = new 생성자();

2. 연관성 있는 저장공간과 기능을 한곳에 모아 편리하게 관리할 수 있는 것

클래스와 객체
	class : 똑같은 무엇인가를 계속 만들어 낼 수 있는 설계도(자동차 설계도, 붕어빵 틀)
	객체(Object) : 클래스에 의해서 만들어진 결과물(자동차, 붕어빵)


클래스 선언
class 클래스명{
	변수; // 멤버 변수, 필드, 전역변수
	메소드(){}; // 멤버 메소드
	// class를 구성하는 모든 변수, 모든 메소드를 합쳐서 클래스의 멤버라고 함
}

생성자
클래스명 객체명 = new 생성자(); -> new는 객체가 들어갈 저장공간을 메모리에 할당
1. 객체를 생성할 때 실행되는 코드가 작성되어 있다
2. 주로 해당 클래스의 필드(멤버변수)를 초기화 하는 목적으로 사용

기본생성자
public 클래스명(){}
: 클래스 선언 시 자동(묵시적 생성자)으로 선언되며, 사용자가 직접 생성자를 선언(명시적 생성자)하게 되면 자동으로
만들어 주지 않는다

this
- 객체 자기 자신을 의미한다 -> 객체 자신의 주소값을 담고 있다

class Student{
	// 필드(전역 변수, 멤버 변수)
	String name;
	int age;
	// 생성자
	Student(String name, int age){
		this.name = name;
		this.age = age;
	}
}

this()
자기자신의 생성자를 의미한다
생성자 내부에서만 사용가능하며 최상단에 작성해야함


1. Static

static : 정적인 <-> 동적인
자바의 저장공간이 프로그램이 시작되고 끝날때까지 유지된다면 => 정적이다
프로그램 실행도중 생겨나거나 사라질 수 있다면 => 동적이다

main 메소드의 코드가 한줄씩 실행 => new를 통해 객체를 생성하는 코드가 있다면
실행도중에 그 객체를 생성함 => 객체 저장공간, 이 객체는 동적으로 생성되는 것이다
=> 객체가 생성되는 heap 영역을 동적 메모리 영역

실행과 만들어져 있는 static 저장공간은 프로그램이 종료 되어야만 메모리에서 사라진다
static이 붙은 저장공간은 method 영역이다

static 키워드 종류
- 멤버 변수(필드)의 종류
	static이 붙은 변수 : static 변수, 클래스 변수, 정적 변수, ...
	static이 안붙은 변수 : 인스턴스(Instance)변수

- 멤버 메소드의 종류
	static이 붙은 메소드 : static 메소드, 클래스 메소드, 정적 메소드, ...
	static이 안붙은 메소드 : 인스턴스(Instance)메소드


1. static 변수, static 메소드는 객체를 생성하지 않아도 사용 가능하다
	클래스명.멤버명; 객체명.멤버명;
2. static 변수는 객체와 상관없이 하나의 저장공간이므로 모든 객체가 공유해서 사용한다
	즉, 모든 객체에서 공통으로 사용되는 변수에 static을 붙이는 것을 고려한다

3. static 메소드는 인스턴스 멤버를 사용할 수 없다
   static 메소드가 메모리에서 사용준비가 끝나도 인스턴스 멤버는 객체를 생성하기 전까지는 
   사용할 수 없기 때문이다

4. 클래스명.멤버명으로 사용하는 것을 권장한다 -> 인스턴스 멤버와 구별해주기 위해서



상속
1. 기존에 선언된 클래스의 멤버(생성자는 제외)를 새롭게 만들 클래스에서
   사용하고 싶을 때 상속을 받고 새로운 기능만 추가한다(확장한다)
2. 부모 클래스를 먼저 선언하고 자식 클래스를 선언해야한다
3. class Parents{
	Parents 멤버;
	// 필드, 전역변수, 멤버변수
	int num1;
	int num2;
	void method01(){}
	// 필드 : 2개
	// 메소드 : 1개
}

class Child extends Parents{
	Parents, Child 멤버;

	int childNum1;
	// int num1;
	// int num2;

	void childMethod01(){}

	필드 : 1개 + 2개(부모 클래스에서 상속받음)
	메소드 : 1개 + 1개(부모 클래스에서 상속)
}

Parents : 부모 클래스, 슈퍼 클래스, 기반 클래스, 상위 클래스
Child : 자식 클래스, 서브 클래스, 파생 클래스, 하위 클래스

** 주의사항
	자바에서 상속은 단일상속만 지원한다(자식 클래스는 1개의 부모 클래스만을 가진다)
